OpenCLI 架构总览

AI-Native CLI Hub — 双引擎架构 · 70+ 适配器 · 浏览器桥接

🚀 入口层 (Entry Layer)
📦
主入口
src/main.ts
CLI 启动入口，负责环境初始化、CLI 发现、插件加载、补全处理和更新检查
PATH 修复 CLI 发现 插件加载 补全处理 更新检查
⚙️
CLI 框架
src/cli.ts
Commander.js 程序初始化，内置命令注册（list, validate, explore, daemon 等）
Commander 初始化 内置命令 参数解析
命令注册 & 发现
⚡ 引擎层 (Engine Layer)
📋
注册中心
src/registry.ts
全局命令注册表，管理所有 CLI 命令的元数据、策略和参数定义
Strategy 枚举 CliCommand 接口 registerCommand() getRegistry()
🔍
发现引擎
src/discovery.ts
扫描 YAML/TS 适配器文件，支持 manifest 快速路径和懒加载优化
YAML 解析 TS 模块扫描 Manifest 快速路径 兼容垫片
🎯
执行引擎
src/execution.ts
命令执行核心，处理参数验证、浏览器会话管理、超时控制和懒加载
参数验证 浏览器会话 超时控制 懒加载 生命周期钩子
🌉
Commander 适配
src/commanderAdapter.ts
Registry → Commander 桥接，处理命令注册、参数绑定和输出渲染
命令注册 参数绑定 输出渲染 错误处理
🎨
输出格式化
src/output.ts
统一输出格式支持：table、json、yaml、markdown、csv
Table 表格 JSON 输出 YAML 输出 Markdown CSV
⏱️
运行时管理
src/runtime.ts
浏览器工厂选择、超时管理、会话生命周期控制
BrowserFactory browserSession() runWithTimeout() 环境变量
双引擎驱动
🔌 适配器层 (Adapter Layer)
📄
YAML Pipeline 引擎
src/pipeline/
声明式数据管道，支持 fetch、map、filter、limit、download 等步骤
executor.ts registry.ts template.ts steps/fetch.ts steps/transform.ts steps/browser.ts steps/intercept.ts steps/tap.ts steps/download.ts
💻
TypeScript 适配器
src/clis/ (70+ 站点)
编程式适配器，支持复杂页面交互、写操作和自定义逻辑
中文平台 全球社交 内容/媒体 AI 工具 新闻/财经 Electron 应用
🔧
外部 CLI Hub
src/external.ts
注册、自动安装和透传执行外部 CLI 工具（gh, docker, vercel 等）
external-clis.yaml 自动安装 透传执行 用户自定义
🧩
插件系统
src/plugin.ts
插件安装/卸载/验证，支持 Git 源和生命周期钩子
install/uninstall Git 源支持 生命周期钩子 兼容性检查
浏览器操作 & 数据获取
🌐 连接层 (Connection Layer)
🖥️
Browser Bridge 模式
src/browser/mcp.ts
通过本地 Daemon 和 Chrome 扩展与浏览器通信，复用登录态
自动启动 Daemon WebSocket 连接 IPage 实现 会话管理
🔌
CDP 模式
src/browser/cdp.ts
直接通过 Chrome DevTools Protocol 连接，支持 Electron 应用控制
WebSocket 连接 Runtime.evaluate Page.enable 事件监听
📄
Page 抽象
src/browser/page.ts
实现 IPage 接口，统一浏览器操作，将所有操作转换为 Daemon 命令
goto() evaluate() snapshot() click() typeText() getCookies() screenshot()
👻
反检测 (Stealth)
src/browser/stealth.ts
修补浏览器指纹，防止被网站检测为自动化工具
navigator.webdriver window.chrome 插件列表伪造 Error stack 清理
📸
DOM 快照引擎
src/browser/dom-snapshot.ts
智能 DOM 裁剪，为 LLM 生成可访问性快照，注入 data-opencli-ref
可见性检测 交互性检测 广告过滤 增量 Diff Shadow DOM iframe 提取
🔧
DOM 操作助手
src/browser/dom-helpers.ts
生成在页面上下文中执行的 JS 代码，用于点击、输入、等待等操作
clickJs() typeTextJs() pressKeyJs() waitForTextJs() autoScrollJs() networkRequestsJs()
🔗
Daemon 客户端
src/browser/daemon-client.ts
与本地 Daemon 通信的 HTTP 客户端，发送命令并接收结果
sendCommand() isDaemonRunning() isExtensionConnected() 重试机制
HTTP POST /command ↔ WebSocket ↔ chrome.* API
🧩 Chrome Extension 层
📡
Service Worker
extension/src/background.ts
扩展后台脚本，管理 WebSocket 连接、命令分发和窗口隔离
WS 连接管理 命令分发 窗口隔离 空闲超时 重连策略
🔍
CDP 执行引擎
extension/src/cdp.ts
通过 chrome.debugger API 执行 JS，管理调试器附件状态
chrome.debugger.attach Runtime.evaluate 断点禁用 URL 验证
📋
协议定义
extension/src/protocol.ts
定义 Daemon 和 Extension 之间的通讯协议类型
Command 接口 Result 接口 Action 类型 常量定义
🛠️ 辅助工具 (Utilities)
🔍
探索工具
src/explore.ts
智能 API 发现，自动滚动、抓包、分析响应、推断能力
🏥
诊断工具
src/doctor.ts
浏览器连通性检测，Daemon 状态检查，自动修复建议
❌
错误系统
src/errors.ts
统一 CliError 层次结构，Unix 风格退出码，友好错误提示
📝
日志系统
src/logger.ts
结构化日志输出，支持不同级别和格式化
🔄
拦截器
src/interceptor.ts
生成 fetch/XHR 拦截代码，用于 Pipeline 的 intercept 步骤
📊
分析模块
src/analysis.ts
URL 模式分析、字段角色检测、能力推断

🔄 完整请求流程 (以 bilibili hot 为例)

1. CLI 解析命令
→
2. execution.ts 判断需要浏览器
→
3. BrowserBridge.connect()
→
4. 启动 Daemon (如未运行)
→
5. 等待 Extension WS 连接
→
6. Page.goto() 导航
→
7. Daemon HTTP POST /command
→
8. Extension chrome.tabs.create()
→
9. 注入 stealth 脚本
→
10. 执行适配器 fetchJson()
→
11. 输出格式化
→
12. 窗口空闲 30s 后关闭
